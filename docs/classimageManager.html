Detailed Description

Adapter class for CIMG library.

The imageManager is the class which will abstract most of the individual functionalities of the Cimg library in a more user-friendly readable fashion.

For the purposes of this project, each "wall" will have its own image manager.

◆ removeLeak()

void imageManager::removeLeak ( )

special deletion of pictures which plugs a memory leak this funciton is needed because of the way that imageManager is implemented. when chooseFirst() is called on the first wall, it returns a vector of pointers to picture objects, being imageManager::failed_Placements. However, when the second wall has chooseFirst called, then imageManager::failed_Placements are never deleted if there are picture objects that could not fit on the second imageManager. So, this function is called to delete imageManager::failed_Placements after the second wall has had chooseFirst() called.

void imageManager::spiralRecur ( bool  up,
bool  right,
int  stop 
)
private

recursive function which implements spiralPlacement()

Parameters
up- keeps track of if the current corner attempting to be placed at is at the top or not. True- is one of the top corners false - is one of the bottom corners
right- keeps track of if the current corner attempting to be placed at is on the right or not. True- is one of the corners on the right side. false - is one of the corners on the left side.
stop- the pixel which the previous recurrence call stopped at. ex. if in top left corner, and moving right, then if the function stops at pixel 800 because of placing a picture, stop becomes 800.
