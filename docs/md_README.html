External Libraries Used:

CImg:

Download and install instructions are on the official webiste! You are going to need to install X11 on your computer in order for CImg to work fully. We very heavily suggest running on Linux, as installation almost always goes much more smoothly.

Here's how we use it in our repo:

the typical way of using CImg involves some confusing code, sometimes needing really long lines in order to display an image. Our project includes the "imageManager" class, which is an adapter class that we use extensively in our program to add photos and sort photos on our wall. for example, this code written using only CImg places a picture onto a 100x100 window and displays it.

{C++}
CImg background<unsigned char>(100,100,1,3,0);
CImg picture<unsigned char>("../pic.jpeg");
background.draw_image(0,0,picture,100);
CImgDisplay Display(100,100,"Display",1);
Display.display(background);

Obviously there are a lot of numbers in these funcitons which stay the same, and do not change for the purposes of placing pictures on a wall. So, with our imageManager class, this becomes a lot more readable and intuitive.

{C++}
imageManager manager(100,100)
manager.placeImage(0,0,picture("../pic.jpeg"));
manager.displayImages();

More information on the above can be found in our Doxygen Documentation!

Command Line Arguments

The program takes in 5 arguments with the first being a directory path (string) and the remaining being pixel dimensions (integer). Argument Format:

"PATH_TO_IMAGE_FOLDER" "WALL_ONE_WIDTH" "WALL_ONE_HEIGHT" "WALL_TWO_WIDTH" "WALL_TWO_HEIGHT"

The image folder will contain only pictures. THE PICTURES SHOULD BE IN .JPEG FORMAT. The name of the picture should represent the price of that particular image when it is placed on the wall. ex: "400.jpeg","299.jpeg" would be worth $400 and $299 respectively. All wall widths and heights should be in units of pixels. output implementation information

Output

The output of the program will be as follows:

Six .jpeg files which represent all of the walls. There are 2 walls per algorithm, and the names will be as follows:

For the Smallest First Algorithm:

  • "smallWallOne.jpeg"
  • "smallWallTwo.jpeg"

For the Expensive First algorithm:

  • "expensiveWallOne.jpeg"
  • "expensiveWallTwo.jpeg"

For the Heuristic First Algorithm:

  • "heuristicWallOne.jpeg"
  • "heuristicWallTwo.jpeg"

One .txt file which will show all the properties of each of the walls

The name will be: "wallOutputFile.txt" This file WILL NOT be in .csv format, and will instead have headers and a lines below them showing which algorithm the information is for.

The properties which are tracked, and which will be outputted are:

  • The total value contained on wall one
  • The total value contained on wall two
  • The amount of wasted space of wall one
  • The amount of wasted space of wall two
  • the total profit across both walls

Algorithms Implemented

Our algorithm uses a "spiral" like strategy to place images, filling the outside edges first before working towards the middle. This same spiral strategy is used for all three algorithms that we have implemented, and we simply change the priority level of each picture. A picture with higher priority will be in a queue to be placed far earlier than a picture that has a low priority. Information about implementation is here Our Priority organizations are as follows:

  • Expensive First - Inserts the most expensive images first onto the wall.
  • Smallest First - Inserts the smallest images onto the wall first
  • Heuristic - Uses weightings and other algorithmic evaluations to prioritize the insertion of images with higher value/square pixel
